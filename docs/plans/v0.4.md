# v0.4 plan — Supervisor

## Task v0.4-supervisor — `src/supervisor.js`
`createSupervisor({ config, inject, sleep, notify, log, now })` →
`{ handleTransient(), handleLimit(text), signalError(), busy, limitResumes }`.

- `handleTransient()`: for each backoff step (parsed via `parseDuration`): `sleep(step)`,
  reset error flag, settle-inject (`sleep(injectDelayMs)` if >0, then `inject(phrase+'\r')`),
  log retry, `sleep(healObserveSeconds)`; if no `signalError` arrived → resumed (log+notify, stop);
  else next step. All steps fail → gave-up (log+notify).
- `handleLimit(text)`: maxResumes cap check; `target = parseReset(text, now()) ?? now()+fallback`;
  `sleep(target-now + grace)`; settle-inject; `limitResumes++`; resumed (log+notify).
- `signalError()`: marks the error recurred (consulted inside the observe window).
- `busy` guard: one recovery at a time; re-entrant calls return `'busy'`.

## Tests (>=8) — deterministic via a manual fake clock (controllable sleep)
heal-on-first; never-heal→5 injects→gave-up; heal-on-2nd→2 injects; limit waits reset+grace then
1 inject; limit fallback wait when unparseable; maxResumes cap → gave-up no sleep; busy guard;
inject payload = phrase+CR; injectDelayMs adds a settle sleep; resume logs+notifies.
